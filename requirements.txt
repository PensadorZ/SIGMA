# ADVERTENCIA CRÍTICA — NO QUITAR ESTA RESTRICCIÓN DE VERSIÓN:
#
# El stack SIGMA-FE usa Langfuse v2 autoalojado (ghcr.io/langfuse/
# langfuse:2 en docker-compose.yml) — deliberado, para evitar la
# dependencia de ClickHouse que exige la v3+.
#
# langfuse>=3 (el SDK Python) usa una API nativa de OpenTelemetry que
# envía trazas a la ruta /api/public/otel — ESA RUTA SOLO EXISTE EN
# SERVIDORES LANGFUSE >=3.22.0. Contra un servidor v2 real, esa ruta
# responde 404 (confirmado con curl directo, no es un problema de
# configuración ni de puerto — la ruta simplemente no existe en la v2).
#
# Si alguna vez actualizas esto a langfuse>=3, DEBES también subir el
# servidor Docker a v3.22+, lo cual reintroduce ClickHouse. Esa es una
# decisión de arquitectura completa, no un simple `pip install --upgrade`.
#
# ---
#
# CRITICAL WARNING — DO NOT REMOVE THIS VERSION CONSTRAINT:
#
# The SIGMA-FE stack uses self-hosted Langfuse v2 (ghcr.io/langfuse/
# langfuse:2 in docker-compose.yml) — deliberately, to avoid the
# ClickHouse dependency required by v3+.
#
# langfuse>=3 (the Python SDK) uses a native OpenTelemetry API that
# sends traces to the /api/public/otel route — THAT ROUTE ONLY EXISTS
# ON LANGFUSE SERVERS >=3.22.0. Against a real v2 server, that route
# returns 404 (confirmed with a direct curl — not a config or port
# issue, the route simply doesn't exist in v2).
#
# If you ever upgrade this to langfuse>=3, you MUST also upgrade the
# Docker server to v3.22+, which reintroduces ClickHouse. That's a full
# architecture decision, not a simple `pip install --upgrade`.

langfuse<3
